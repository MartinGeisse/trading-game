
Recipes als enum wird schnell Probleme machen. Besser: Als Objekte anlegen in einer als "configuration phase".

-> Builder pattern
-> Später DSL
-> sollte aber im Code stehen, nicht in der DB (hat sonst was von Code-in-DB, auch wenn die Unterscheidung da
	schwammig ist)

Entsprechend sollten auch mögliche Actions zentral gespeichert, bei Programmstart per builder erzeugt werden und
daraus dann auch die Links in der GUI. Das gilt auch dann, wenn die Actions teilweise ausgeblendet werden, bis
der Spieler dafür "reif" ist. Die Preconditions wiederum können auch in dieser init-phase gebaut werden.

Das ist so eine art "game definition" und dies sollte dann per Guice an alle Programmteile weitergegeben werden.
